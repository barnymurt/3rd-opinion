# Logical Reasoning Domain

Analyzes AI responses for logical fallacies, coherence, and argument validity.

## What to Look For

### Common Fallacies
- **Ad Hominem**: Attacking source instead of argument
- **Straw Man**: Misrepresenting the position
- **False Dichotomy**: Presenting only two options when more exist
- **Slippery Slope**: Unwarranted chain of consequences
- **Circular Reasoning**: Conclusion restates premise
- **Hasty Generalization**: Broad claim from limited evidence
- **Post Hoc**: Assuming causation from correlation

### Coherence Issues
- Internal contradictions within the response
- Unsupported assertions
- Non-sequitur conclusions
- Missing logical connections

### Validity Checks
- Are the premises stated clearly?
- Do conclusions follow from premises?
- Are there hidden assumptions?

## Trigger Indicators

Look for these signals in AI response:
- Words: "always", "never", "everyone", "nobody", "must", "certainly"
- Patterns: absolutes, generalizations, causal claims without evidence
- Structure: opinion presented as fact, correlation presented as causation

## Analysis Questions

1. What is the main claim or conclusion?
2. What evidence supports it?
3. Are there logical gaps or unsupported assertions?
4. What alternative explanations exist?
5. What would need to be true for this to be invalid?

## Domain Weight Factors

Increase weight when:
- Response makes definitive claims
- Contains causal statements
- Uses absolute language (always/never/must)
- Presents opinion as fact

Decrease weight when:
- Response is purely informational
- Contains multiple viewpoints
- Uses hedged language appropriately

## Integration Notes

This domain combines well with:
- Fact-checking (verify claims)
- Bias detection (identify framing)
- Perspective diversity (alternative viewpoints)
